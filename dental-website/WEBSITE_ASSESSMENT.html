Um ecossistema digital de alta performance desenvolvido para captação de leads e marketing de conteúdo no setor odontológico.

Resumo Executivo

Este projeto transcende a definição de um site institucional comum. Trata-se de uma aplicação web completa (Full-Stack Application) equipada com um Sistema de Gerenciamento de Conteúdo (CMS) personalizado.

A arquitetura foi desenhada meticulosamente para SEO (Search Engine Optimization), utilizando renderização no servidor (SSR) para garantir indexação imediata pelo Google, crucial para a estratégia de captação de pacientes da clínica.

02. Modelagem de Dados (Schema)

A estrutura de dados (definida em schema.prisma) revela a inteligência de negócios embutida na aplicação. O sistema não apenas armazena textos, mas gerencia relacionamentos complexos de marketing.

Modelo Função Estratégica
User & Auth Sistema robusto de autenticação baseado em sessões (NextAuth v5), com suporte a múltiplos níveis de permissão (Admin, Editor, Viewer).
Content Engine O modelo Post é o coração do SEO. Inclui campos dedicados para meta-tags (seoTitle, seoDescription) e métricas de engajamento (views, readingTime). Suporta fluxo editorial (Rascunho → Agendado → Publicado).
Lead Intelligence O modelo Lead vai além de um formulário de contato. Ele captura parâmetros UTM (utmSource, utmCampaign), permitindo rastrear exatamente qual anúncio (Instagram, Google) gerou o paciente.

05. Plano de Evolução (Roadmap)

Para elevar a plataforma ao nível "Enterprise", recomendamos as seguintes ações estratégicas:

  • 1. Estratégia de Persistência de Mídia (Crítico) Atualmente, o modelo Media armazena metadados, mas o armazenamento físico de arquivos em containers (Railway) é efêmero.
    Ação Recomendada: Integrar AWS S3 ou Cloudflare R2 para armazenamento definitivo de imagens.
  • 2. Notificações em Tempo Real
    Ação Recomendada: Implementar gatilhos de e-mail (via Resend) para alertar a recepção instantaneamente quando um novo Lead for cadastrado. "Tempo de resposta" é vital para conversão médica.
  • 3. Dashboard de Business Intelligence (BI)
    Ação Recomendada: Transformar os dados brutos de Leads em gráficos visuais no Admin. Visualizar "Pacientes por Canal de Aquisição" justificará o investimento em marketing.
  • 4. Homepage Dinâmica
    Ação Recomendada: Migrar os textos da Homepage (atualmente hardcoded) para o banco de dados, permitindo que a clínica altere ofertas e textos sem intervenção de desenvolvedores.
